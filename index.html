Installation in 3 Schritten

  • Öffne getskales.app/download und lade die passende Version.
  • Installiere wie gewohnt (Windows: .exe, macOS: .dmg, Linux: AppImage/DEB).
  • Starte SKALES – der Assistent führt dich durch Sprache, Provider und Modell.

API-Key (OpenRouter)

Empfohlener Einstieg: ein OpenRouter-Key schaltet viele Modelle frei.

  • Account bei openrouter.ai anlegen.
  • Key unter /settings/keys erzeugen.
  • In SKALES unter Settings → Providers → OpenRouter eintragen.
  • Modell wählen und chatten.

Features & Ziele

Lokaler KI-Agent ohne komplexes Dev-Setup – ideal für Lernende, Lehrkräfte und Entwickler:innen.

  • Lokale App: Läuft als Desktop-App, nicht nur im Browser.
  • Multi-Provider: OpenRouter, Ollama, Custom Endpoints u.a.
  • Modellwechsel: Direkt im Chat per Picker oder /model.
  • Kein Docker nötig: Installer herunterladen, starten, fertig.
  • Datenhoheit: Mit Ollama bleiben Daten lokal.
  • Flexibel: Für Coding, Orga, Recherchen und Automationen.
  • Quellen offen: Source-available auf GitHub (BSL‑1.1).

Modelle & Provider

Drei Wege zu deinen eigenen Modellen.

Variante A: OpenRouter (Cloud)

  • Key unter openrouter.ai/settings/keys erstellen.
  • In SKALES Settings → Providers → OpenRouter öffnen.
  • Key eintragen, Modell auswählen, fertig.

Variante B: Ollama (lokal)

  • Ollama installieren (siehe docs.ollama.com).
  • Modell laden, z.B. ollama pull llama3.
  • In SKALES Settings → Providers → Ollama aktivieren.
  • Modelle per Refresh einlesen und auswählen.

Variante C: Custom Endpoint

  • Settings → Providers → Custom Endpoint öffnen.
  • Basis-URL deiner OpenAI-kompatiblen API eintragen.
  • Ggf. API-Key hinterlegen, Modellliste aktualisieren, Modell wählen.
